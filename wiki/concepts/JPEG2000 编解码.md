# JPEG2000 编解码

**JPEG2000**（简称 J2K）是 DCP 和 IMF 母版制作的底层图像压缩编码标准。Transkoder 提供业界最高的 J2K GPU 加速编解码性能。

## 在 Transkoder 中的作用

- **DCP 视频轨道**：所有 DCP 视频 MXF 必须使用 J2K 编码（Interop 或 SMPTE 标准）
- **IMF 视频轨道**：IMF 视频资产基于 J2K 压缩
- **性能核心**：Transkoder 的 GPU 加速 J2K 引擎是系统的关键竞争力

## Transkoder 的 J2K 特性

| 特性 | 说明 |
|------|------|
| **GPU 加速** | 编解码完全 GPU 加速，多 GPU 可线性扩展 |
| **32 位浮点** | 32 位/通道浮点处理精度 |
| **无损/有损** | 支持 JPEG2000 无损和有损压缩 |
| **自适应编码** | 支持 PSNR 和比特率目标定位的自适应 J2K 编码 |
| **性能** | 4K DCP J2K 编码 100+ fps |
| **位率分析** | 内置 PSNR 分析工具 |

## 压缩选项（DCP）

根据 DCP 标准，J2K 编码需要配置：
- **比特率目标**：通常 250 Mbps（2K）/ 500 Mbps（4K）
- **质量优先** vs **比特率优先** 策略
- **自适应模式**：根据画面复杂度动态分配带宽

## 相关概念

- DCP 的 J2K 编码在 Finalize 窗口的 DCP Encode 选项卡配置
- IMF 支持更灵活的 J2K 配置（Lossy/Lossless）
