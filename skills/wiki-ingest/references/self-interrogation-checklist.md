# Self-Interrogation Checklist

Run this checklist for every non-trivial ingest, especially manuals, platform guides, process docs, and large technical sources.

## Required Questions

1. What are the most likely future user questions about this source?
2. Which parts of the source describe stable workflows or repeatable procedures?
3. Which terms, settings, fields, windows, parameters, or constraints deserve dedicated concept treatment?
4. Which answers would still force a future query to reopen raw sections if I do not compile them now?

## Required Action

Do not leave the answers in scratch reasoning only.

Turn them into one or more of:

- workflow pages
- concept pages
- source-page question lists
- question seeds or durable question pages

## Warning Sign

If the output is mostly one source page, one domain page, and a few broad summaries, you probably did not interrogate the source deeply enough.
